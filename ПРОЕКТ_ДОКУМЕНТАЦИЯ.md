# ДОКУМЕНТАЦИЯ ПРОЕКТА EZY: CYBER - ЛИСТ ПЕРСОНАЖА

## ОБЩАЯ СТРУКТУРА ПРОЕКТА

Проект представляет собой интерактивный лист персонажа для настольной ролевой игры EZY: Cyber. Это одностраничное веб-приложение (SPA) с сохранением данных в localStorage.

---

## АРХИТЕКТУРА ФАЙЛОВ

### 1. **index.html** - Главный HTML файл
- Содержит всю разметку интерфейса
- Подключает все JavaScript файлы в строгом порядке (важно!)
- Определяет структуру секций: Общая информация, Навыки, Оружие, Броня, Дека, Снаряжение, Препараты, Транспорт и т.д.
- Содержит модальные окна для различных функций

**Порядок подключения скриптов (критически важен!):**
1. `data.js` - библиотеки данных и объект state
2. `professional-skills.js` - профессиональные навыки
3. `utils.js` - вспомогательные функции
4. `armor.js` - система брони
5. `transport.js` - транспорт
6. `drugs.js` - препараты
7. `shops.js` - магазины
8. `deck.js` - деки
9. `weapons.js` - оружие
10. `gear.js` - снаряжение
11. `scripts.js` - основной функционал
12. `fence.js` - скупщик
13. `bargain-purchase.js` - система торга

## ЦЕНТРАЛЬНЫЙ ОБЪЕКТ СОСТОЯНИЯ (state)

**Расположение:** `data.js`, строка 2178

Глобальный объект `state` содержит ВСЕ данные персонажа:

```javascript
const state = {
    // Основная информация
    characterName: '',
    experiencePoints: 0,
    roleplayPoints: 0,
    avatar: '',
    
    // Характеристики (1-20)
    stats: {
        WILL: 5, INT: 5, DEX: 5, BODY: 5,
        REA: 5, TECH: 5, CHA: 5
    },
    
    // Удача
    luck: { current: 1, max: 1 },
    
    // Производные характеристики
    awareness: { current: 50, max: 50 },
    reputation: 0,
    
    // Здоровье и ресурсы
    health: { current: 25, max: 25 },
    money: 3500,
    
    // Навыки
    skills: [],                    // Обычные навыки
    professionalSkills: [],        // Профессиональные навыки (максимум 4)
    bargainEnabled: true,         // Включён ли торг
    
    // Дека
    deck: { name, operative, grid, memory, version },
    deckRam: { current, max },
    deckPrograms: [],
    deckChips: [],
    deckGear: [],
    decks: [],                    // Коллекция дек
    
    // Оружие и боеприпасы
    weapons: [],
    ammo: [],
    weaponModules: [],
    
    // Броня
    armor: {
        head: { os, type, armorName, ... },
        body: { os, type, armorName, ... },
        arms: { os, type, armorName, ... },
        legs: { os, type, armorName, ... },
        reinforcement: { head, body, arms, legs },  // Укрепления костной системы
        description: ''
    },
    armorInventory: [],           // Инвентарь купленной брони
    titanicArmorConnected: false,  // Подключена ли титаническая броня
    
    // Снаряжение
    gear: [],
    load: { current, max },       // Нагрузка
    
    // Препараты
    drugs: [],
    
    // Имущество
    property: {
        vehicles: [],             // Транспорт
        housing: [],              // Жилье
        commercialProperty: []    // Коммерческая недвижимость
    },
    
    // Киберимпланты
    implants: {
        head: { installed, parts: { main } },
        arms: { installed, parts: { wristLeft, wristRight, ... } },
        legs: { installed, parts: { footLeft, footRight, ... } },
        spine: { installed, parts: { cervical, thoracicLeft, ... } },
        organs: { installed, parts: { main } },
        neuromodule: { installed, parts: { main } }
    },
    installedModules: [],         // Установленные модули имплантов
    
    // Модули транспорта
    vehicleModules: [],
    
    // Скупщик
    fenceShop: {
        itemPrices: {}            // Кэш цен для предметов
    },
    
    // Критические травмы
    criticalInjuries: [],
    
    // Предыстория
    backstory: '',
    backstoryGraphs: {},
    
    // Логи и заметки
    rollLog: [],
    notes: '',
    counters: [],
    transactionLog: [],
    
    // Настройки интерфейса
    uiSettings: {},
    sectionSizes: {},
    layoutOrder: [],
    collapsedItems: {}            // Состояние сворачивания элементов
}
```

## ВЗАИМОСВЯЗИ МЕЖДУ МОДУЛЯМИ

### 1. **data.js** - Центр данных
**Зависимости:** Нет (базовый файл)

**Экспортирует:**
- `state` - глобальный объект состояния
- `STANDARD_SKILLS` - библиотека стандартных навыков
- `PROFESSIONAL_SKILLS_LIBRARY` - библиотека профессиональных навыков
- `WEAPONS_LIBRARY` - библиотека оружия
- `WEAPON_MODULES_LIBRARY` - библиотека модулей оружия
- `ARMOR_LIBRARY` - библиотека брони
- `CYBERIMPLANTS_LIBRARY` - библиотека киберимплантов
- `DRUGS_LIBRARY` - библиотека препаратов
- `VEHICLES_LIBRARY` - библиотека транспорта
- `VEHICLE_MODULES_LIBRARY` - библиотека модулей транспорта
- `HOUSING_OPTIONS` - варианты жилья
- `COMMERCIAL_PROPERTY_OPTIONS` - коммерческая недвижимость
- `DECK_LIBRARY` - библиотека дек
- `PROGRAMS_LIBRARY` - библиотека программ для дек
- `DECK_GEAR_LIBRARY` - снаряжение для дек
- `GEAR_LIBRARY` - библиотека снаряжения

**Используется:** ВСЕМИ модулями

---

### 2. **utils.js** - Вспомогательные функции
**Зависимости:** `data.js`

**Ключевые функции:**
- `saveState()` - сохранение в localStorage
- `loadState()` - загрузка из localStorage (с инициализацией полей)
- `scheduleSave()` - автосохранение с debounce
- `updateUIFromState()` - обновление всех UI элементов из state
- `exportData()` / `importData()` - экспорт/импорт JSON
- `getThemeColors()` - получение цветов темы
- `showModal()`, `showConfirmModal()`, `showPromptModal()` - модальные окна
- `canBargain()` - проверка доступности торга
- `getProfessionalSkillLevel(name)` - получение уровня профессионального навыка
- `formatMoney()` - форматирование денег
- `generateId()` - генерация уникальных ID

**Используется:** ВСЕМИ модулями для сохранения, модалок, форматирования

---

### 3. **scripts.js** - Основной функционал
**Зависимости:** `data.js`, `utils.js`, все остальные модули

**Ключевые функции:**
- `calculateDerivedStats()` - расчёт производных характеристик (ВОС, Скорость, Скорость атаки)
- `calculateAndUpdateHealth()` - расчёт здоровья (ТЕЛО*5 + ВОЛЯ*3)
- `updateAwarenessMax()` - обновление максимальной осознанности (УМ*10)
- `updateDerivedStats()` - обновление всех производных характеристик
- `recalculateLoadFromInventory()` - пересчёт нагрузки из снаряжения
- `rollSkillCheck(skillIndex)` - бросок проверки навыка
- `addToRollLog()` - добавление в лог бросков
- `renderSkills()` - отрисовка навыков
- `updateMoneyDisplay()` - обновление отображения денег
- `initLoadingScreen()` - загрузочный экран
- `initAvatarUpload()` - загрузка аватара
- `generateBackstory()` - генерация предыстории
- `addCriticalInjury()` - добавление критической травмы

**Взаимодействия:**
- Вызывает `renderWeapons()`, `renderGear()`, `renderTransport()` и т.д. из других модулей
- Использует функции из `utils.js` для сохранения и модалок
- Обновляет `state` при изменении характеристик

---

### 4. **armor.js** - Система брони
**Зависимости:** `data.js`, `utils.js`

**Ключевые функции:**
- `showArmorShop()` - магазин брони
- `buyArmor()` - покупка брони
- `equipArmorPiece(armorId)` - надевание брони из инвентаря
- `removeEquippedArmor(bodyPart)` - снятие брони
- `updateArmorDisplay()` - обновление отображения брони (с учётом укреплений)
- `calculateArmorPenalties()` - расчёт штрафов от брони
- `updateArmorPenalty()` - обновление штрафов и UI
- `renderArmorInventory()` - отрисовка инвентаря брони
- `repairArmorPiece(armorId)` - починка брони

**Взаимодействия:**
- Изменяет `state.armor` и `state.armorInventory`
- Вызывает `updateDerivedStats()` из `scripts.js` для обновления скорости
- Вызывает `calculateAndUpdateHealth()` при подключении титанической брони
- Использует `getThemeColors()` из `utils.js`
- Использует `scheduleSave()` для автосохранения

**Особенности:**
- Броня хранится в `state.armor[bodyPart]` (head, body, arms, legs)
- Укрепления костной системы в `state.armor.reinforcement[bodyPart]`
- При надевании брони учитывается приоритет: броня > укрепление
- Штрафы применяются к характеристикам: СКО, РЕА, ЛВК, ПЗ

---

### 5. **weapons.js** - Система оружия
**Зависимости:** `data.js`, `utils.js`, `gear.js` (для нагрузки)

**Ключевые функции:**
- `showWeaponShop()` - магазин оружия
- `showWeaponModulesShop()` - магазин модулей оружия
- `buyRangedWeapon()` / `buyMeleeWeapon()` - покупка оружия
- `buyWeaponModule()` - покупка модуля оружия
- `installWeaponModule()` - установка модуля на оружие
- `renderWeapons()` - отрисовка оружия
- `renderAmmo()` - отрисовка боеприпасов
- `showAmmoShop()` - магазин боеприпасов
- `showShotgunShootingModal()` - стрельба из дробовика

**Взаимодействия:**
- Изменяет `state.weapons`, `state.ammo`, `state.weaponModules`
- Модули оружия могут храниться в `state.gear` до установки
- При установке модуля он удаляется из `state.gear` и добавляется в `weapon.modules[]`
- Использует `recalculateLoadFromInventory()` из `scripts.js` для обновления нагрузки
- Использует `addToRollLog()` для логирования стрельбы

**Особенности:**
- Оружие имеет слоты для модулей (`weapon.slots`)
- Модули могут занимать 1 или 2 слота
- Дробовики имеют специальную систему зарядки (2 типа патронов)
- Боеприпасы хранятся отдельно в `state.ammo`

### 6. **gear.js** - Система снаряжения
**Зависимости:** `data.js`, `utils.js`, `scripts.js` (для нагрузки)

**Ключевые функции:**
- `showGearShop()` - магазин снаряжения
- `pickupGear()` - подобрать предмет с пола
- `renderGear()` - отрисовка снаряжения
- `recalculateAndUpdateLoad()` - пересчёт нагрузки
- `updateLoadDisplay()` - обновление отображения нагрузки
- `resetLoad()` - сброс нагрузки

**Взаимодействия:**
- Изменяет `state.gear` и `state.load`
- Модули оружия и транспорта могут храниться в `state.gear` до установки
- Вызывает `updateDerivedStats()` при изменении нагрузки (влияет на скорость)
- Велосипед - особая сущность: при переключении "Катить" появляется в транспорте

**Особенности:**
- Нагрузка влияет на скорость перемещения (штраф при перегрузке)
- Каждый предмет имеет поле `load` (нагрузка)
- Максимальная нагрузка = ТЕЛО * 5

### 8. **transport.js** - Система транспорта
**Зависимости:** `data.js`, `utils.js`, `weapons.js` (для оружия в модулях)

**Ключевые функции:**
- `showTransportShop()` - магазин транспорта
- `showTransportModulesShop()` - магазин модулей транспорта
- `buyVehicle()` - покупка транспорта
- `buyVehicleModule()` - покупка модуля транспорта
- `renderTransport()` - отрисовка транспорта
- `recalculateVehicleStats(vehicleId)` - пересчёт характеристик транспорта
- `showVehicleTrunk()` - багажник транспорта
- `showVehicleModulesManagement()` - управление модулями

**Взаимодействия:**
- Изменяет `state.property.vehicles`
- Модули транспорта могут храниться в `state.gear` (велосипед) или `vehicle.trunk[]`
- При установке модуля он удаляется из багажника/снаряжения
- Модули с оружием могут содержать оружие в `module.weapons[]`
- При продаже транспорта оружие из модулей возвращается в `state.weapons`
- Использует `getProfessionalSkillLevel('Друг семьи')` для скидки на транспорт

**Особенности:**
- Транспорт имеет багажник (`vehicle.trunk[]`) для хранения предметов
- Модули могут требовать определённые категории транспорта
- Велосипед - особая сущность, связанная с `state.gear`
- Скорость велосипеда = (Атлетика + ТЕЛО) / 2 + 5

### 9. **deck.js** - Система дек (кибердеки)
**Зависимости:** `data.js`, `utils.js`, `professional-skills.js`

**Ключевые функции:**
- `showDeckShop()` - магазин дек
- `showProgramShop()` - магазин программ
- `showDeckGearShop()` - магазин снаряжения для дек
- `buyDeck()` - покупка деки
- `buyProgram()` - покупка программы
- `renderDeckPrograms()` - отрисовка программ
- `renderDeckChips()` - отрисовка щепок памяти
- `renderDeckGear()` - отрисовка снаряжения дек
- `renderDecksCollection()` - коллекция дек
- `installChipOnDeckTarget()` - установка щепки на деку

**Взаимодействия:**
- Изменяет `state.deck`, `state.deckPrograms`, `state.deckChips`, `state.deckGear`, `state.decks`
- Использует `checkDeckVisibility()` из `professional-skills.js` для показа/скрытия блока Дека
- Блок Дека показывается только если есть навык: Чёрный хакер, Белый хакер или Программист
- Программы устанавливаются на деку через `program.installedDeckId`

**Особенности:**
- Основная дека в `state.deck`, дополнительные в `state.decks[]`
- Программы имеют ОЗУ (`ram`), которое вычитается из `state.deckRam.current`
- Щепки памяти увеличивают память деки
- Снаряжение дек улучшает характеристики деки

### 10. **shops.js** - Магазины (жилье, недвижимость)

### 11. **fence.js** - Скупщик (продажа вещей)
**Зависимости:** `data.js`, `utils.js`, `bargain-purchase.js`

**Ключевые функции:**
- `showFenceShop()` - открытие магазина скупщика
- `getSellableItems()` - получение всех предметов для продажи
- `getFencePrice(item)` - расчёт цены скупщика (20-70% от каталожной)
- `initiateSaleWithPrice()` - начало продажи
- `startBargaining()` - начало торга
- `performBargainCheck()` - проверка навыка Торг
- `calculateFinalPrice()` - расчёт итоговой цены с учётом Решалы
- `executeSale()` - выполнение продажи
- `removeItemFromInventory()` - удаление предмета из инвентаря
- `updateFenceShop()` - обновление ассортимента скупщика

**Взаимодействия:**
- Собирает предметы из: `state.weapons`, `state.ammo`, `state.gear`, `state.deckPrograms`, `state.property.vehicles`, `state.property.housing`, `state.decks` и т.д.
- Использует `canBargain()` из `utils.js` для проверки навыка Торг
- Использует `getProfessionalSkillLevel('Решала')` для модификатора цены
- При продаже добавляет деньги в `state.money`
- Удаляет предметы из соответствующих массивов в `state`

**Особенности:**
- Скупщик НЕ покупает: броню, транспорт (если есть предметы в багажнике), медикаменты
- Цена скупщика случайная (20-70% от каталожной), сохраняется в `state.fenceShop.itemPrices`
- При успешном торге цена увеличивается, при провале - уменьшается
- Уровень Решалы влияет на модификатор цены (±10% до ±100%)

### 12. **bargain-purchase.js** - Система торга при покупке
**Зависимости:** `data.js`, `utils.js`, все модули с функциями покупки

**Ключевые функции:**
- `showPurchaseBargainChoice()` - выбор: торговаться или купить сразу
- `startPurchaseBargaining()` - начало торга
- `performPurchaseBargainCheck()` - проверка навыка Торг
- `calculatePurchaseDiscount()` - расчёт скидки по уровню Решалы
- `showPurchaseSuccess()` - показ успешной покупки со скидкой

**Взаимодействия:**
- Обёртывает функции покупки из других модулей: `buyGear`, `buyRangedWeapon`, `buyMeleeWeapon`, `buyProgram`, `buyWeaponModule`, `buyVehicleModule`, `buyDrug`, `buyVehicle`, `buyDeck`, `buyDeckGear`
- Использует `canBargain()` из `utils.js`
- Использует `getProfessionalSkillLevel('Решала')` для скидки
- Передаёт итоговую цену (со скидкой) в оригинальную функцию покупки

**Особенности:**
- Торг доступен только если есть навык "Торг" и `state.bargainEnabled === true`
- Скидка по уровню Решалы: 1-2: 10%, 3-4: 20%, 5-6: 30%, 7: 40%, 8-9: 50%, 10: 100% (бесплатно!)
- При провале проверки Торг - покупка по полной цене

### 13. **professional-skills.js** - Профессиональные навыки
**Зависимости:** `data.js`, `utils.js`

**Ключевые функции:**
- `showProfessionalSkillsModal()` - модальное окно выбора навыка
- `selectProfessionalSkill(skillId)` - выбор навыка
- `addProfessionalSkillToSlot(skillId, level)` - добавление навыка в слот
- `removeProfessionalSkillFromSlot(slotIndex)` - удаление навыка
- `updateProfessionalSkillLevelNew()` - обновление уровня навыка
- `renderProfessionalSkills()` - отрисовка навыков
- `checkDeckVisibility()` - проверка видимости блока Дека

**Взаимодействия:**

- Использует `PROFESSIONAL_SKILLS_LIBRARY` из `data.js`
- Вызывает `checkDeckVisibility()` при изменении навыков
- Используется другими модулями через `getProfessionalSkillLevel(name)`:
  - "Друг семьи" - скидка на транспорт (10% за уровень)

## КЛЮЧЕВЫЕ ВЗАИМОСВЯЗИ И ЗАВИСИМОСТИ

### Цепочка обновлений при изменении характеристик:

1. **Изменение базовой характеристики** (например, ТЕЛО):
   - `scripts.js`: `calculateAndUpdateHealth()` → обновляет `state.health.max`
   - `scripts.js`: `updateDerivedStats()` → обновляет ВОС, Скорость, Скорость атаки
   - `scripts.js`: `recalculateLoadFromInventory()` → обновляет нагрузку
   - `armor.js`: `updateArmorPenalty()` → обновляет штрафы от брони (если есть)
   - `utils.js`: `scheduleSave()` → автосохранение

2. **Надевание брони**:
   - `armor.js`: `confirmEquipArmor()` → обновляет `state.armor[bodyPart]`
   - `armor.js`: `updateArmorPenalty()` → рассчитывает штрафы
   - `scripts.js`: `updateDerivedStats()` → применяет штрафы к скорости
   - `scripts.js`: `calculateAndUpdateHealth()` → применяет штраф к здоровью (титаническая броня)

3. **Покупка предмета**:
   - Модуль магазина → проверка денег → списание `state.money`
   - `bargain-purchase.js`: обёртка торга (если доступен)
   - Добавление предмета в соответствующий массив (`state.weapons`, `state.gear` и т.д.)
   - Обновление нагрузки (если снаряжение)
   - `utils.js`: `scheduleSave()` → автосохранение
   - `utils.js`: `addToRollLog()` → логирование покупки

4. **Установка модуля на оружие/транспорт**:
   - Модуль удаляется из `state.gear` или `vehicle.trunk[]`
   - Модуль добавляется в `weapon.modules[]` или `vehicle.modules[]`
   - Пересчёт характеристик (для транспорта)
   - Обновление нагрузки
   - `utils.js`: `scheduleSave()`

## КРИТИЧЕСКИЕ МОМЕНТЫ ДЛЯ ИСПРАВЛЕНИЯ ОШИБОК

### 1. **Порядок загрузки скриптов:**
- НЕ МЕНЯТЬ порядок в `index.html`!
- `data.js` должен загружаться первым
- `scripts.js` должен загружаться последним (после всех модулей)

### 2. **Инициализация state:**
- При добавлении нового поля в `state` нужно:
  - Добавить в `data.js` (начальное значение)
  - Добавить инициализацию в `loadState()` (utils.js)
  - Добавить в `exportData()` и `performClearAllData()` (scripts.js)

### 3. **Обновление UI:**
- После изменения `state` ВСЕГДА вызывать соответствующую функцию рендеринга
- Использовать `scheduleSave()` для автосохранения
- Для производных характеристик вызывать `updateDerivedStats()`

### 4. **Взаимодействие модулей:**
- Модули НЕ должны напрямую вызывать функции друг друга (кроме `utils.js`)
- Использовать глобальные функции через `window.functionName`
- Проверять существование функции перед вызовом: `if (typeof functionName === 'function')`

### 5. **Работа с массивами:**
- При удалении элемента нужно обновить `collapsedItems` через `updateCollapsedItemsAfterRemoval()`
- При изменении индексов нужно пересчитать индексы в `collapsedItems`

### 6. **Модальные окна:**
- Всегда блокировать скролл: `document.body.style.overflow = 'hidden'`
- Восстанавливать скролл при закрытии: `document.body.style.overflow = ''`
- Использовать `addModalKeyboardHandlers()` для обработки Enter/Escape

## ЧАСТЫЕ ОШИБКИ И РЕШЕНИЯ

### Ошибка: "Cannot read property 'X' of undefined"
- **Причина:** Поле не инициализировано в `state`
- **Решение:** Добавить инициализацию в `loadState()` (utils.js)

### Ошибка: "Function is not defined"
- **Причина:** Функция вызывается до загрузки модуля
- **Решение:** Проверить порядок загрузки скриптов или использовать проверку `typeof`

### Ошибка: "UI не обновляется после изменения данных"
- **Причина:** Не вызвана функция рендеринга
- **Решение:** Вызвать соответствующую `render*()` функцию после изменения `state`

### Ошибка: "Данные не сохраняются"
- **Причина:** Не вызван `scheduleSave()`
- **Решение:** Добавить `scheduleSave()` после изменения `state`

### Ошибка: "Штрафы от брони не применяются"
- **Причина:** Не вызван `updateArmorPenalty()` после изменения брони
- **Решение:** Вызвать `updateArmorPenalty()` и `updateDerivedStats()`
